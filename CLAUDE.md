# 통합보장분석 자동화 프로젝트

## 프로젝트 개요
메리츠화재 등 보장분석 화면에서 전체 복사한 텍스트를 받아, `통합보장분석` 엑셀 템플릿의
**'통합보장 분석(전)' 시트**를 자동으로 채우는 작업. 상세 규칙은 아래 파일 참조.

## 핵심 파일
- `./보장분석_자동화_규칙_v12.md` — 파싱/매핑/서식 규칙 전체 (최신 버전을 항상 우선 참조. 파일명 버전 숫자가 더 높은 파일이 있으면 그것을 사용)
- `./통합보장분석5_4.xlsx` — 빈 템플릿 (건드리지 말고 복사해서 작업)
- 결과물: `./output/{고객명}_통합보장분석.xlsx`

## 작업 원칙 (모든 명령어 공통)
1. **작업 전 반드시 규칙 파일 최신본을 다시 읽는다.** (매번 새 세션에서 규칙이 갱신됐을 수 있음)
2. **파일 전체를 재저장하지 않는다.** openpyxl 등으로 다시 쓰면 서식이 깨진다.
   반드시 xlsx를 zip으로 풀어 `xl/worksheets/sheet*.xml`의 대상 셀 값만 직접 문자열 치환/삽입하고
   다시 zip으로 묶는 **XML 레벨 주입** 방식을 사용한다.
3. Total(E열) 수식은 절대 건드리지 않는다. 대신 워크북 XML의 `calcPr`에 `fullCalcOnLoad="1"`을 설정하고
   `xl/calcChain.xml`은 제거(또는 workbook.xml.rels/[Content_Types].xml에서 참조 제거)하여
   파일을 열 때 자동 재계산되게 한다.
4. F75, F80, F84, F88의 깨진 `#REF!` 수식은 값 주입 시 함께 삭제한다.
5. **애매한 케이스는 절대 임의 판단하지 않고 사용자에게 질문한다.** 규칙 파일 규칙 15, 21 참조.
6. 파일 생성 전 반드시 **검증표(파싱 결과 표)를 사용자에게 먼저 보여주고 확인을 받는다.**
7. 파일 생성 후 반드시 다음 두 가지를 채팅에 (시트 아님) 표시한다:
   - 조회 한계 고지 문구 (규칙 23)
   - 계약별 약관 원문 링크 (규칙 22)
8. 새로운 담보명이나 케이스가 발견되어 규칙이 갱신되면, `보장분석_자동화_규칙_vN.md`를
   다음 버전(N+1)으로 사용자와 상의 후 갱신한다.

## 사용법
사용자가 메리츠 등 보장분석 화면에서 복사한 텍스트를 붙여넣으며 `/보장분석`을 호출하면,
`.claude/commands/보장분석.md`에 정의된 절차를 그대로 따른다.
