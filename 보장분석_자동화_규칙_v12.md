# 통합보장분석 시트 자동 작성 규칙 (v12)

## 목적
메리츠화재 보장분석 화면에서 전체 복사한 텍스트를 붙여넣으면, '통합보장분석' 엑셀 템플릿의 **'통합보장 분석(전)' 시트**를 자동으로 채운다.
(증권 파일 직접 분석 자동화는 추후 진행. 당분간 메리츠 보장분석 텍스트 기준으로 케이스를 학습하며 이 규칙 파일을 업데이트한다.)

## 작업 방식 (필수)
- 파일 전체를 변환/재저장하지 않는다. **XML 레벨에서 대상 셀 값만 주입**하여 원본의 서식(색상, 병합, 조건부서식, 숨김행)을 100% 보존한다.
- Total(E열) 수식은 건드리지 않고, 파일 열 때 자동 재계산되도록 `fullCalcOnLoad` 설정 + calcChain 제거.
- 결과 파일명: `{고객명}_통합보장분석.xlsx`

## 시트 구조
- 행: 담보 항목 고정 (아래 매핑표 기준)
- G열~P열: 계약 슬롯 (최대 10건). **계약일(가입일)이 빠른 순서대로 G열부터 오른쪽으로 순차 기재** (붙여넣은 순서가 아님)
- E열: Total 자동 합산 (수정 금지)
- A1: 제목 → `{고객명} 고객님 통합보장 분석` — **굵게, 40pt, 맑은 고딕 리치텍스트로 고정**
- A4: 생년월일 (날짜) → C3 보험나이 자동 계산됨

## 입력 텍스트 구조 (메리츠 보장분석 복사본)
계약 블록마다:
1. 회사명 / 상품명 / 증권번호
2. `정상YYYY.MM.DD ~ YYYY.MM.DD` (계약일 ~ 만기)
3. `(매월납, N년)` = 납입기간, 월보험료
4. `납입완료 금액` / `납입예정 금액`
5. 정액형 보장 표: 보장구분 / 보장명 / 보장금액 / 보장상태
6. 실손형 보장 표: 실손구분 / 보장명 / 보장금액 / 보장기간 / 상태
※ 조회 한계: **정액형 담보는 생명보험 2000.04, 손해보험 2002.12 이후 계약에 한하여 조회됨** → 규칙 23의 고지 문구 참조.

## 핵심 규칙
1. **보장상태 필터**: `정상`만 반영. `해지`, `만기`는 제외.
2. **80%이상 후유장해**: 무시 (3%↑ 행에 합산하지 않음).
3. **통원의료비**: 외래 + 처방조제료 **합산**하여 기입 (예: 25만+5만 → 30).
   - **상해+질병 통합 실손** (예: '상해+질병 입원의료비'): 질병(16·17행)과 상해(18·19행) **양쪽에 각각 기재**.
4. **교통상해사망**: 무시. '상해사망' 행에는 일반상해사망만 기재.
5. **미반영 담보**: 시트에 대응 행이 없으면 그냥 버림 (별도 정리 불필요).
6. **보험기간(10행) / 납입기간(9행) 표기** — 판단 순서 중요:
   - 만기일을 보험나이로 환산 (보험나이 = DATEDIF(생년월일−6개월, 만기일, "y"))
   - **0순위 — 종신형 판정**: 만기일이 `9999.12.31` 등 종신형이면 → 납입기간 `N년납`, 보험기간 `종신/만기`.
   - **1순위 — 세만기 판정**: 만기 보험나이가 80세/90세/100세 등 세만기(통상 60세 이상)이면 → 납입기간 `N년납` (예: 20년납, 25년납, 30년납), 보험기간 `N세/만기` (예: 100세/만기).
     - ⚠️ **개별 담보명에 '(갱신형)'이 붙어 있어도 계약 만기가 세만기면 무조건 세만기로 표기한다.** 담보 단위 갱신 여부는 계약 단위 보험기간 판정에 영향을 주지 않음.
   - **2순위 — 세만기가 아닌 경우** (보험기간이 10/20/30년 등 기간형): 납입기간에 '납' 없이 주기만 표기 (예: 10년, 20년, 30년), 보험기간은 아래로 구분:
     - 해당 계약 블록 텍스트 **어디에든** '갱신' 문구가 보이면 (상품명, 담보명의 '(갱신형)' 포함) → 보험기간 `갱신`
     - '갱신' 문구가 전혀 안 보이면 → 보험기간 `만기`
   - 모든 계약에 공통으로 붙는 안내 문구 `갱신 등이 반영되지 않은 단순 예상 보험료`는 판단 근거에서 **제외**.
   - 세만기 여부나 갱신/만기 구분이 애매하면 사용자에게 질문.
7. **보장금액 단위**: 만원 (1억 → 10000, 5,000만원 → 5000).
   - **갱신형 빨간색 표기 (확정)**: ① 담보명에 '(갱신형)' 등 갱신 표기가 붙은 담보는 그 **보장금액 셀을 글꼴 색 빨강**으로 기재. ② **계약 자체가 갱신형**(보험기간이 `갱신`인 계약)이면 그 **계약 열의 보장금액 전체를 빨강**으로 기재. (XML 주입 시 기존 셀 서식을 복제하고 글꼴 색만 빨강으로 바꾼 스타일을 추가 적용, 테두리·배경·숫자서식 보존. 계약정보 행(회사명·기간·보험료 등)은 검정 유지, 보장금액 행만 대상.)
   - **밑줄 표기**: 아래 두 경우 금액 숫자에 밑줄(_)을 표기한다. ① 골절진단비가 '치아파절제외'만 있는 경우(규칙 11), ② 입원일당이 '4일 이상' 조건부인 경우(규칙 24).
8. **보험료(11, 13~15행)**: 원 단위. 총납입보험료(13행) = 납입완료+납입예정 수식.
9. **회사명(6행)**: 아래 순서로 확정한다.
   - ① 붙여넣기 텍스트의 회사명과 상품명이 일치하면 그 회사명 사용.
   - ② **회사명 오표기 주의**: 원문에 '삼성생명' 등으로 표기돼 있어도 실제 계약이 아닐 수 있음(복사 오류 빈발). 회사명이 없거나 상품명과 불일치하거나 의심스러우면 **상품명을 웹 검색하여 실제 판매 보험사를 확정** 후 기재 (예: '차도리' 브랜드 = 한화손해). 검색으로도 애매하면 사용자에게 질문.
   - ③ **표준 표기**: 아래 회사명 표준표를 따른다.

### 회사명 표준표 (콜센터 목록 기준)
- 생명보험사: 삼성생명, 교보생명, DB생명, 동양생명, 라이나생명, 메트라이프, 미래에셋생명, 한화생명, 신한생명, KB생명, 흥국생명, NH농협생명, CHUBB, 푸르덴셜, 푸본현대생명, DGB생명, 우체국보험, 오렌지라이프, KDB생명, ABL생명, 하나생명
- 손해보험사: 삼성화재, 현대해상, 메리츠화재, 흥국화재, AIG, AXA다이렉트, 그리고 **다음 7개사는 '보험'을 빼고 4글자로 표기**: `DB손해`, `KB손해`, `한화손해`, `롯데손해`, `MG손해`, `농협손해`, `하나손해`
- 사용자가 특정 계약의 회사명을 직접 지정/정정하면 그 표기가 최우선 (예: 이필우 건의 '농협생명').

10. **보험종류(7행)**: 상품명 전체가 아닌 유형으로 표기. 키워드 우선순위:
    운전자 > 화재 > 치매 > 간병·요양 > 종신 > 건강·종합
    → 운전자보험 / 화재보험 / 치매보험 / 간병보험 / 종신보험 / 건강보험
    추가 유형: 실손 단독 상품 → `단독실손`, 암보험 → `암보험`, 어린이보험 → `어린이보험`
    - **생명보험사 계약에서 일반사망이 주계약이면 → `종신보험`으로 표기.**
11. **골절/5대골절/화상/깁스 (61~64행)**: 담보명에 '진단비'로 표기된 것만 기재. 골절수술비, 화상수술비 등 수술비류는 기재하지 않음.
    - 예외: **깁스치료비는 깁스진단비로 간주하여 그대로 기재** (64행). 단, **신깁스치료비(급여)는 미반영**.
    - **골절진단비 복수**: '치아파절제외'와 '치아파절포함'이 모두 있으면 **합산** 기재 (예: 20+10 → 30). '치아파절제외'만 있으면 금액 숫자에 **밑줄(_) 표기**.
12. **생명보험사 사망 합산**: 회사명에 **'생명' 또는 '라이프'가 들어간 생명보험사** (교보생명, 삼성생명, 라이나생명 등) 계약으로 상해사망과 질병사망이 모두 있는 경우:
    - ⚠️ 판단 기준은 **최종 확정된 회사명** (사용자가 회사명을 정정한 경우 정정된 이름 기준). 붙여넣기 원문의 회사명과 상품명이 불일치하면 규칙 9에 따라 먼저 회사명을 확정한 뒤 이 규칙을 적용한다.
    - 두 금액이 같으면: 합치지 말고 그 금액을 **일반사망(69행)** 에 기재 (예: 상해 3천 + 질병 3천 → 일반사망 3,000).
    - 두 금액이 다르면: **작은 금액을 일반사망**에, **초과분(차액)을 상해사망 행**에 기재 (예: 상해 5천 + 질병 3천 → 일반사망 3,000 + 상해사망 2,000).
    - 질병사망이 상해사망보다 큰 케이스는 통상 없음. 만약 발견되면 임의 처리하지 말고 **사용자에게 질문**.
    - 회사명에 **'손해', '화재', '해상'이 들어간 손해보험사** (메리츠화재, 삼성화재, 현대해상, 한화손해 등)는 합산하지 않고 **있는 그대로** 상해사망/질병사망 행에 각각 기재. (예: 한화손해에 질병사망 1,000 + 상해사망 100 → 질병사망 행 1,000, 상해사망 행 100. 어느 쪽도 누락하거나 합치지 않음)
    - 공제 등 회사명만으로 생보/손보 구분이 애매하면 사용자에게 질문.
13. **운전자보험 벌금**: 벌금 담보가 여러 개면 (예: 벌금 3천만 + 벌금(대물) 5백만) **가장 큰 금액 하나만** 벌금(75행)에 기재.
14. **종수술비 (50~52행)**: 종별 금액 중 **가장 큰 종(최고종) 기준 금액**을 기재.
    - 1~3종 → 최고종 금액을 '1~3종수술비'(50행)에
    - 1~5종 → 최고종 금액을 '1~5종수술비'(51행)에
    - 1~7종 → 최고종 금액을 '1~7종수술비'(52행)에
    - 1~8종, 1~9종 → 최고종 금액을 52행에 넣되, **행 이름(C52)을 '1~8종수술비' 또는 '1~9종수술비'로 수정**
15. **애매하거나 헷갈리는 케이스는 임의 판단하지 말고 반드시 사용자에게 질문한다.** 새 기준이 확정되면 규칙 파일에 반영한다. 질문 대상이 된 담보라도 답을 받기 전까지는 **누락시키지 말고 보류 목록으로 명시**한다.
16. 숨김행이 열려도 무방 (사용자가 직접 다시 숨김).
17. **교통사고처리지원금 복수 시**: 처리지원금 담보가 여러 개면 (예: 처리지원금 2억 + 6주미만 진단 1,000만) **가장 큰 금액 하나만** 77행에 기재. (벌금 규칙 13과 동일한 방식)
18. **특정부위/특정조건 한정 담보 미반영**: `외모특정상해수술비` 등 특정 부위/조건 한정 수술비는 일반 상해/질병수술비 행에 기재하지 않고 **미반영(버림)**. **상해성(외상성) 뇌출혈 진단비/수술비**도 뇌혈관(41~45행)에 기재하지 않고 미반영 (해당 행은 질병성 뇌혈관질환 기준). 일반 담보인지 한정 담보인지 애매하면 사용자에게 질문.
19. **원문에 없는 값 생성 금지**: 시트에 기입하는 모든 담보명과 금액은 붙여넣은 텍스트에 실제로 존재하는 것이어야 한다. 원문에 없는 담보나 금액을 추측·창작해서 기입하는 것은 절대 금지.
20. **잔여 오류 수식 정리**: 원본 템플릿의 F75, F80, F84, F88에 있는 깨진 수식(#REF!)은 값 주입 시 반드시 함께 삭제한다.
21. **기입 전 검증 절차 (필수)**: 파일을 만들기 전에 아래를 수행한다.
    - 파싱 결과를 표로 정리해 사용자에게 먼저 보여준다: 계약별 기본정보 + 담보→시트 행 매핑 + 금액.
    - 원문의 '정상' 담보 개수와 (기입 + 규칙상 제외) 담보 개수를 대조해 **1건도 누락이 없는지 확인**하고, 규칙에 따라 제외한 담보는 제외 사유와 함께 명시한다. 특히 사망·후유장해처럼 규칙 분기가 있는 담보는 이중 확인.
    - 사용자 확인 후 파일을 생성한다.
22. **약관 링크 제공 (파일 생성 후 필수)**: 엑셀 파일을 전달한 뒤, **채팅 메시지에** 계약별 약관 원문 링크를 따로 정리해 단다 (시트에는 넣지 않음).
    - 대상은 **약관 원문**이다. 상품요약서는 대체물이 아니며, 약관을 못 찾았을 때만 '약관 아님'을 명시하고 보조로 제공할 수 있다.
