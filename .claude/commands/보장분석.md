# 통합보장분석 자동 작성

사용자가 아래에 메리츠화재 등 보장분석 화면에서 전체 복사한 텍스트를 붙여넣었다:

$ARGUMENTS

## 실행 절차 (반드시 순서대로, 단계를 건너뛰지 말 것)

### 0단계 — 규칙 최신본 로드
1. 프로젝트 폴더에서 `보장분석_자동화_규칙_v*.md` 중 **버전 숫자가 가장 높은 파일**을 찾아 전체를 읽는다.
2. 빈 템플릿 `통합보장분석*.xlsx`를 찾아 `xl/worksheets/sheet*.xml`, `xl/sharedStrings.xml`,
   `xl/styles.xml` 구조를 확인한다. (행 번호는 규칙 파일의 담보 매핑표 기준)

### 1단계 — 파싱
붙여넣은 텍스트를 계약 블록 단위로 분리하고, 규칙 파일의 아래 항목을 계약별로 판정한다:
- 회사명 (규칙 9, 표준표 적용, 오표기 의심 시 상품명 웹 검색)
- 보험종류 (규칙 10)
- 계약일 → 계약일 빠른 순으로 G열부터 슬롯 배정
- 납입기간/보험기간 (규칙 6: 종신형 0순위 → 세만기 1순위 → 기간형 2순위)
- 보험료 3종 (월보험료/납입완료/납입예정)
- '정상' 상태인 정액형·실손형 담보만 추출 (해지/만기 제외)
- 각 담보를 매핑표의 시트 행 번호에 배정, 규칙 2~4·11·13·14·17~19·24·25 등 특수 규칙 적용
- 갱신형 여부(담보 단위/계약 단위) → 빨간색 표기 대상 표시
- 밑줄 표기 대상(골절 파절제외 단독, 입원일당 조건부) 표시

### 2단계 — 검증표 제시 (파일 생성 전 필수, 절대 생략 금지)
사용자에게 아래를 표로 보여주고 확인을 요청한다:
- 계약별 기본정보 (회사명/상품명/보험종류/계약일/납입·보험기간/보험료)
- 담보 → 시트 행 매핑 + 금액 (빨간색/밑줄 대상 표시)
- 원문 '정상' 담보 총 개수 vs (기입 개수 + 제외 개수) 대조, 제외 사유 명시
- **판단이 애매한 담보는 별도 "질문/보류 목록"으로 명시** — 임의로 매핑하지 말 것.
  필요하면 약관 원문을 먼저 검색해 근거를 확보한 뒤 질문한다 (규칙 22 순서 참조).

사용자가 확인/수정하기 전까지 3단계로 진행하지 않는다.

### 3단계 — XML 레벨 셀 주입
1. 템플릿 xlsx를 작업 디렉토리에 복사 (원본 보존).
2. zip으로 해제 후 `sheet*.xml`에서 대상 셀(`<c r="G16">` 등)을 찾아 값만 치환.
   - 셀이 없으면 새로 생성하되 기존 style index(`s="..."`)는 그대로 유지.
   - 갱신형 대상 셀은 styles.xml에 폰트색만 빨강으로 바꾼 새 스타일을 추가하고 그 셀의 `s` 속성만 교체
     (테두리/배경/숫자서식은 기존 스타일 복제로 보존).
   - 밑줄 대상 셀도 동일한 방식으로 밑줄 폰트 스타일 추가.
3. A1 제목: `{고객명} 고객님 통합보장 분석` (굵게, 40pt, 맑은 고딕 리치텍스트) 셀 텍스트만 교체.
4. A4 생년월일 입력.
5. F75, F80, F84, F88의 `#REF!` 수식 제거.
6. workbook.xml의 `calcPr`에 `fullCalcOnLoad="1"` 설정, `calcChain.xml` 제거 및 관련 참조 정리.
7. 다시 zip으로 묶어 `output/{고객명}_통합보장분석.xlsx` 생성.
8. 생성 후 openpyxl 등으로 값/서식이 의도대로 들어갔는지 읽어서 검증 (읽기 전용 확인, 재저장 금지).

### 4단계 — 전달 및 고지
파일을 전달하며 채팅에 다음을 반드시 포함한다:
1. 조회 한계 고지 문구 (규칙 23 원문 그대로)
2. 계약별 약관 원문 링크 (규칙 22 절차대로 검색, 못 찾으면 사유와 대체 안내)

## 주의
- 원문에 없는 담보/금액을 절대 만들어내지 않는다 (규칙 19).
- 애매하면 언제나 질문한다 (규칙 15). 새 판단 기준이 확정되면 사용자와 상의해 규칙 파일을
  다음 버전으로 갱신한다.
